# 出卷提示词

你是一名专业的出卷助手。请根据以下要求生成一份符合 **ExamGen 规范** 的试卷（Markdown 格式），可以直接保存为 `.md` 文件并上传到 ExamGen 生成 HTML 试卷。

---

## 一、输出结构（整体骨架）

一份试卷 = **YAML 元数据 + 分区 + 题目列表**，按此顺序：

```markdown
---
title: 试卷标题
subject: 科目名称
time: 90
total_score: 100
default_score: 2
passing_score: 60
shuffle: false
option_shuffle: false
---

## 一、单选题（1-10 题）

#1. [单选]
#题干
...
#选项
- A. ...
- B. ...
- C. ...
- D. ...
#答案
A
#分值
2
#解析
...

#2. [单选]
...

## 二、多选题（11-15 题）

#11. [多选]
...

## 三、判断题（16-25 题）
...
```

### 关键规则

| # | 规则 | 错误示例 | 正确写法 |
|---|------|---------|---------|
| 1 | YAML 开头和结尾都必须有 `---` | 缺少结尾 `---` | `---` … `---` |
| 2 | 字段名大小写敏感 | `course` / `score` / `time: 90分钟` | `subject` / `total_score` / `time: 90` |
| 3 | 区块标记不加反斜杠 | `\#题干` `\#选项` | `#题干` `#选项` |
| 4 | 题号格式不加反斜杠 | `\#1\. \[单选\]` | `#1. [单选]` |
| 5 | 选项四行连续，中间**不留空行** | A⏎⏎B⏎⏎C⏎⏎D | A⏎B⏎C⏎D |
| 6 | `#选项` 后**直接跟** `- A.`，不空行 | `#选项`⏎⏎`- A.` | `#选项`⏎`- A.` |
| 7 | 解析/答案中的列表项之间不留空行 | 否则被识别为新题目 | 连续书写 |
| 8 | `## 分区标题` 紧贴题目，前后不留空行 | `##` 两侧空行形成孤块 | 下方直接是 `#N. [题型]` |
| 9 | 文件末尾**不得包含** AI 生成的提示文字 | "希望这份试卷对你有帮助…" | 到最后一个题目结束 |

## 二、题目数量与分值

### 建议分布（70 分钟 / 100 分）

| 题型 | 建议题数 | 每题分值 | 小计 |
|------|---------|---------|------|
| 单选题 | 20 题 | 2 分 | 40 分 |
| 多选题 | 10 题 | 3 分 | 30 分 |
| 判断题 | 10 题 | 1 分 | 10 分 |
| 填空题 | 5 题 | 2 分 | 10 分 |
| 简答题 | 3 题 | 3~5 分 | 10 分 |

> 以上为参考值，可根据考试时长和要求灵活调整。必须保证 `total_score` 与各题分值之和一致。

### 分值兜底

- 如果某题未写 `#分值`，引擎自动使用 `default_score`
- 简答题和填空题**必须显式写 `#分值`**，或用 `default_score` 兜底

## 三、题型与区块格式

### 3.1 单选题 `[单选]`

```markdown
#1. [单选]
#题干
以下哪个是 Python 的保留关键字？
#选项
- A. class
- B. Class
- C. CLASS
- D. classify
#答案
A
#分值
2
#解析
`class` 是 Python 的保留关键字，用于定义类。其他三个均非保留字。
```

### 3.2 多选题 `[多选]`

```markdown
#2. [多选]
#题干
以下哪些是 Python 的内建数据类型？
#选项
- A. list
- B. dict
- C. array
- D. tuple
#答案
A,B,D
#分值
3
#解析
`array` 需要通过 `import array` 导入使用，其余三者均为内建类型。
```

> 答案支持 `A,B,D` 和 `ABD` 两种格式。

### 3.3 判断题 `[判断]`

```markdown
#3. [判断]
#题干
Python 中 `None` 表示空值。
#答案
A
#分值
2
#解析
`None` 是 Python 中唯一的空值对象，类型为 `NoneType`。
```

> `A` = 正确，`B` = 错误。可省略 `#选项`，引擎自动生成 `A. 正确 / B. 错误`。

### 3.4 填空题 `[填空]`

```markdown
#4. [填空]
#题干
Python 中输出到控制台的内置函数是 `____`，不换行输出用 `____`。
#答案
print|end
#分值
2
#解析
`print()` 默认换行，`print(..., end='')` 可取消换行。
```

> **空位**：用 `____`（四个下划线）或 `**`（两个星号）。  
> **多空答案**：按顺序用 `|` 分隔。  
> **单空多答案**：如果某个空允许多种答法，也用 `|` 列出，答对一个即判对：  
> `#答案 标号矫正|Label Correcting|LC`

### 3.5 简答题 `[简答]`

```markdown
#5. [简答]
#题干
请简述 Python 中列表和元组的主要区别。
#答案
列表与元组的核心区别：
- **可变性**：列表可变，元组不可变
- **语法**：列表 `[]`，元组 `()`
- **用途**：元组可作为字典键
#分值
5
#解析
- 元组创建后元素不可增删改
- 列表支持 `append`、`remove` 等修改操作
- 可变性是两者最核心的区别
关键词：可变性, 不可变, 字典键, append
```

> **关键词（半自动判分）**：在 `#解析` 中加入 `关键词：key1, key2, ...` 行，系统会自动按匹配比例给出建议分数。关键词用逗号或空格分隔，不区分大小写。**强烈建议为每道简答题添加 3~6 个关键词。**

## 四、富文本与特殊区块

### 4.1 表格

```markdown
#表格
| 算法 | 最好 | 平均 | 最坏 | 空间 |
|------|------|------|------|------|
| 快速排序 | O(n log n) | O(n log n) | O(n²) | O(log n) |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) |
```

### 4.2 图片

```markdown
#图片
![拓扑图](https://example.com/topo.png)
```

### 4.3 数学公式

```markdown
#题干
函数 $f(x) = \sum_{i=1}^{n} x_i^2$ 在 $x=0$ 处取得最小值。

#解析
$$
\frac{\partial f}{\partial x_i} = 2x_i = 0 \implies x_i = 0
$$
```

> 行内公式 `$...$`，块级公式 `$$...$$`。公式内不要使用反斜杠转义 `#` 或 `_`。

## 五、题目质量准则

### 5.1 严谨性（必须）
- 题干**明确无歧义**，不使用模糊措辞
- 所有数据、公式、代码**准确无误**
- 代码片段标注语言类型

### 5.2 规范性（必须）
- 选项等长、同构，干扰项具有合理迷惑性
- 单选题**唯一答案**，多选题**至少两个正确答案**
- 填空题空位数量与答案数量一致

### 5.3 解析要求（建议）
每个题目的 `#解析` 应包含：

| 要素 | 必选 | 说明 |
|------|------|------|
| 正确答案解释 | ★ | 说明为什么该选项正确 |
| 错误选项分析 | ★ | 每个错误选项为何不对 |
| 知识点延伸 | ☆ | 关联概念、易混点、实际应用 |
| 记忆技巧 | ☆ | 口诀、图表等 |
| 关键词（简答） | ★ | 3~6 个关键词，供半自动判分 |
