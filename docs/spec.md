# 题目源文件规范 — ExamGen 格式参考

ExamGen 使用 Markdown 编写试卷。本文档是解析引擎的格式参考，涵盖 YAML 元数据、区块标记、题型格式和常见错误排查。

---

## 一、文件结构

```
---
YAML 元数据
---

## 分区标题（可选）

#N. [题型]
#题干
...
#选项（选择/判断题必填）
...
#答案
...
#分值（可选，不填则用 default_score）
#解析（可选）
...
```

---

## 二、YAML 元数据

### 字段表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | **是** | 试卷标题 |
| `subject` | string | 否 | 科目（注意不是 `course`） |
| `time` | int | 否 | 考试时长（分钟），纯数字如 `90` |
| `total_score` | float | 否 | 试卷总分（注意不是 `score`） |
| `default_score` | float | 否 | 未指定分值时默认分，默认 `1.0` |
| `shuffle` | bool | 否 | `false` — 在每个 `## 分区` 内打乱题目顺序，保持分区相对顺序 |
| `option_shuffle` | bool | 否 | `false` — 打乱选项顺序，答案字母自动映射 |
| `passing_score` | float | 否 | 及格分，填了才显示是否及格 |

### 常见字段错误

| 错误写法 | 正确写法 | 说明 |
|----------|----------|------|
| `course: ...` | `subject: ...` | 字段名错误 |
| `score: ...` | `total_score: ...` | 字段名错误 |
| `time: 90分钟` | `time: 90` | `time` 只接受纯整数 |
| 缺少 `default_score` | `default_score: 2` | 简答/填空无显式分值时兜底 |
| 缺少 `passing_score` | `passing_score: 60` | 不填则不显示是否及格 |

## 三、区块标记

| 标记 | 含义 | 必填 | 说明 |
|------|------|------|------|
| `#题干` | 题目主干 | **是** | 支持 Markdown、LaTeX 公式 |
| `#图片` | 插图 | 否 | `![描述](URL)` |
| `#表格` | 数据表格 | 否 | GFM 表格语法 |
| `#选项` | 选择题选项 | 选择/判断题必填 | `- A. xxx` 四行连续 |
| `#答案` | 参考答案 | **是** | 判分依据 |
| `#分值` | 分值 | 否 | 不填则用 `default_score` |
| `#解析` | 题目解析（知识点背景，不是答案） | 否 | 提交后展示，不要与 `#答案` 内容重复 |

### 通用规则

- 每个标记独占一行，以 `#` 开头，**不加反斜杠转义**
- `#答案` 是判分依据（如 `7|16`），`#解析` 是知识点解释（如"OSI 模型..."），二者职责不同，不要混淆
- 标记后的内容直到下一个标记或题目结束都属于该区块
- 区块内支持完整 Markdown 语法（表格、代码、列表、图片等）
- 每道题第一行固定：`#N. [题型]`，`N` 为序号
- 题型取值：`[单选]` / `[多选]` / `[判断]` / `[填空]` / `[简答]`

## 四、题型格式

### 4.1 单选题 `[单选]`

```
#N. [单选]
#题干
...
#选项
- A. ...
- B. ...
- C. ...
- D. ...
#答案
A
#分值
2
#解析
...
```

- 选项四行连续，`#选项` 后直接跟 `- A.` **不留空行**
- 答案：单个大写字母

### 4.2 多选题 `[多选]`

```
#N. [多选]
#选项
- A. ...
- B. ...
- C. ...
- D. ...
#答案
A,B,D
```

- 答案格式：`A,B,D` 或 `ABD`，至少两个字母
- 其他规则同单选

### 4.3 判断题 `[判断]`

```
#N. [判断]
#题干
...
#答案
A
```

- `A` = 正确，`B` = 错误
- `#选项` **可省略**，引擎自动生成 `A. 正确 / B. 错误`

### 4.4 填空题 `[填空]`

```
#N. [填空]
#题干
...____...
#答案
print|end
```

- 空位用 `____`（四个下划线）或 `**`（两个星号）
- 多个空答案按顺序用 `|` 分隔
- 单空允许多个可接受答案时也用 `|` 列出，答对一个即判对

### 4.5 简答题 `[简答]`

```
#N. [简答]
#题干
...
#答案
...
#分值
5
#解析
...
关键词：key1, key2, key3
```

- 提交后展示参考答案，标记"待评"
- `#解析` 中可包含 `关键词：xxx, yyy` 行（逗号或空格分隔，不区分大小写），系统按匹配比例给建议分

### 4.6 多分区

```
## 一、选择题（1-10 题）
#1. [单选]
...
#10. [单选]

## 二、多选题（11-15 题）
#11. [多选]
...
```

- `## 分区标题` 必须紧贴下一题 `#N. [题型]`，**前后不留空行**

## 五、富文本支持

- **Markdown**：题干/解析/答案支持完整 Markdown（加粗、列表、引用、代码、表格等）
- **数学公式**：行内 `$...$`，块级 `$$...$$`，由 KaTeX 渲染。

> ⚠️ **致命错误：LaTeX 命令必须全部小写！**  
> KaTeX 严格区分大小写，大写命令不会渲染，公式将原样显示为纯文本。  
> **错误**：`\FRAC` `\TIMES` `\APPROX` `\SUM` `\INT` `\LIM` `\PI` `\DIV`  
> **正确**：`\frac` `\times` `\approx` `\sum` `\int` `\lim` `\pi` `\div`  
> 一句话：**只要你按了 Shift 或 CapsLock，公式就废了。**
- **图片**：`![描述](URL)`，支持远程 URL 和本地图片。

### 5.1 图片使用说明

#### 本地图片（推荐：网页上传时附带图片文件）

在 Markdown 中使用相对路径引用图片（如 `_figures/xxx.png`），上传时**同时选择 .md 文件和图片文件**，服务器按**文件名**自动匹配嵌入 base64。

```markdown
![网络拓扑图](_figures/network_reliability.png)
```

只需上传 `network_reliability.png` 即可匹配，不关心目录路径。

> ⚠️ 图片嵌入匹配规则：提取 Markdown 中 `![alt](path)` 的 `path` 部分，取**文件名**（去除目录前缀）与上传图片的文件名对比，一致则嵌入。因此 `_figures/a.png`、`images/a.png`、`a.png` 都会匹配上传的 `a.png`。

#### 远程图片

直接使用 URL，无需上传：

```markdown
![示例图](https://example.com/image.png)
```

#### Base64 内嵌

将图片直接写在 Markdown 文件中（适合小图片）：

```markdown
![图](data:image/png;base64,iVBORw0KGgo...)
```

可使用 `scripts/embed_images.py` 工具将本地图片一键嵌入到 .md 文件中。

## 六、常见错误排查

| # | 现象 | 原因 | 正确做法 |
|---|------|------|----------|
| 1 | Internal Server Error | YAML 缺结尾 `---` | 必须有 `---` 开头和结尾 |
| 2 | 字段不识别 | 写了 `course`/`score`/`time: 90分钟` | `subject`/`total_score`/`time: 90` |
| 3 | 区块解析为空 | `\#题干` `\#选项` | `#题干` `#选项`，不加反斜杠 |
| 4 | 题号不识别 | `\#1\. \[单选\]` | `#1. [单选]` |
| 5 | 选项空白 | `\- A\. xxx` | `- A. xxx` |
| 6 | 选项缺 B/C/D | 选项之间有空行 | 四行连续，中间不空行 |
| 7 | 答案/解析被截断 | 列表项之间有空行 | 列表项之间不留空行 |
| 8 | `##` 被当题目 | `##` 两侧有空行 | `##` 紧贴下一题 |
| 9 | `#选项` 后内容为空 | `#选项` 后留空行 | 直接跟 `- A.` |
| 10 | 区块内容为空 | 标记行后留空行 | 紧跟前一个区块 |
| 11 | 末尾内容误解析 | 末尾有 AI 提示文字 | 全部删除 |
| 12 | 图片不显示（裂图） | 网页端仅上传 .md 未上传图片文件 | 上传时同时选择 .md 和图片文件 |
| 13 | 图片不显示（裂图） | 上传的图片文件名与 Markdown 中的引用名不匹配 | 确保上传的图片文件名（如 `xxx.png`）与 Markdown 中 `![alt](path)` 的 `path` 文件名一致 |
| 14 | 公式原样显示不渲染 | LaTeX 命令用了**大写字母**（如 `\FRAC`、`\TIMES`） | KaTeX 只认小写命令，全部改为小写：`\frac`、`\times`、`\approx`、`\sum` 等 |
